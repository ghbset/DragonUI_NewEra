# compat/ — Modern-API Coverage Matrix (Sprint 0)

Every modern symbol the NewEra **v1** modules (`CharacterPanel`, `Spellbook`, `Talents`,
`QuestFrame`, `MerchantFrame`, `MailFrame`) and their `Core/` dependencies reference, with how
3.3.5a satisfies it after `compat/` loads.

Symbol list generated by:
```
grep -rhoE "C_[A-Za-z]+\.[A-Za-z]+|\bMixin\b|CreateFromMixins|CreateAndInitFromMixin" \
  "/root/downport/DOWNPORT THIS/NewEra/"{CharacterPanel,Spellbook,Talents,QuestFrame,MerchantFrame,MailFrame,Core} \
  | sort | uniq -c | sort -rn
```

**Source legend**
- `ClassicAPI` — when `!!!ClassicAPI` is loaded it already defines this; our shim leaves it.
- `vendored` — `compat/` defines a fresh fallback (used when ClassicAPI absent). Our file owns it.
- `DragonUI` — satisfied by the base DragonUI addon (not by compat/).
- `stub` — `compat/` defines it but it CANNOT answer correctly on 3.3.5a; returns a safe value.
  Recorded at runtime in `NE.compat.stubs`.
- `not-needed-v1` — appears in NewEra source but only in code paths the v1 modules don't hit
  (other namespaces / out-of-scope panels). Listed for completeness; **no shim shipped** unless a
  v1 guard could still reach it.

**Detection rule:** each `compat/` file does `C_X = C_X or {}; if not C_X.Fn then C_X.Fn = … end`,
so a working ClassicAPI impl is never clobbered. `NE.compat.<cap>` booleans
(`timer/mixin/texture/container/item/spell/map`) report which layers guaranteed their symbols.

## Symbols seen in source but NOT shimmed by compat/ (not-needed-v1)

These appear in the grep over Core / v1 dirs but belong to namespaces outside compat/'s charter.
They are either supplied by ClassicAPI directly (if present), handled by another agent's layer, or
only reached by out-of-scope code paths. **No compat/ shim is shipped for these in Sprint 0.**

| Symbol | Disposition | Why not in compat/ |
|---|---|---|
| `C_SpecializationInfo.GetTalentInfo` | not-needed-v1 / Talents-owned | 3.3.5 has talent tabs, not specs; the Talents module downports talent reading against `GetTalentInfo`/`GetTalentTabInfo` directly (its concern, not a generic shim). |
| `C_SpecializationInfo.GetActiveSpecGroup` | not-needed-v1 / Talents-owned | Talent group = dual-spec group on 3.3.5 (`GetActiveTalentGroup`); Talents module maps it. |
| `C_SpecializationInfo.GetSpecializationInfo` | not-needed-v1 / Talents-owned | No spec system on 3.3.5; Talents module synthesizes. |
| `C_EquipmentSet.*` (Create/Save/Use/Delete/Modify/GetEquipmentSetIDs/GetEquipmentSetInfo/GetItemIDs/GetIgnoredSlots/IgnoreSlotForSave/…) | not-needed-v1 / CharacterPanel-owned | Per memory `[3.3.5a addon API gotchas]` the native equip-set API is flaky; CharacterPanel uses the **ItemRack model** in its own SavedVariables, not `C_EquipmentSet`. The v1 panel doesn't call these. |
| `C_NewItems.IsNewItem` / `RemoveNewItem` | not-needed-v1 | "New item" glow is a Classic bag feature; ClassicAPI ships `C_NewItems` if loaded. v1 bag/merchant paths guard or skip it. |
| `C_MerchantFrame.GetNumJunkItems` / `SellAllJunkItems` | not-needed-v1 / MerchantFrame-owned | No native junk API on 3.3.5; MerchantFrame downport iterates bags itself for the sell-junk button. |
| `C_CVar.GetCVarBool` | DragonUI / global | 3.3.5 global `GetCVarBool` exists; if a `C_CVar` wrapper is wanted ClassicAPI provides it. Not a compat/ deliverable. |
| `C_AddOns.IsAddOnLoaded` | global | 3.3.5 global `IsAddOnLoaded` exists; ported call sites use the global. ClassicAPI provides `C_AddOns` if needed. |
| `C_ActionBar.FindSpellActionButtons` | not-needed-v1 | Spellbook drag-to-bar nicety; out of v1 scope. |
| `C_Reputation.*` (GetFactionDataByIndex/SetWatchedFactionByIndex/IsFactionActive/SetFactionActive/GetSelectedFaction) | not-needed-v1 | Reputation pane is not a v1 panel; CharacterPanel v1 ships without rep tab. |
| `C_UnitAuras.GetAuraDataByIndex` | not-needed-v1 | Aura frames not in v1 panel set. |
| `C_FriendList.*` (ShowFriends/GetWhoInfo/GetFriendInfoByIndex) | not-needed-v1 | Social/who is not a v1 panel. ClassicAPI ships `C_FriendList`. |
| `C_GuildInfo.GuildRoster` | not-needed-v1 | Guild pane not in v1. |
| `C_ChatBubbles.GetAllChatBubbles` | not-needed-v1 | Chat-bubble skinning, unrelated to v1 panels. |

## Runtime self-report

`NE.compat` exposes:
- capability bools: `classicAPI, timer, mixin, texture, container, item, spell, map`
- `NE.compat.stubs` — array of `{ sym, why }` for every symbol that could only be stubbed
  (`C_Spell.GetSpellPowerCost`, `C_Map.GetBestMapForUnit`, `C_Map.GetMapInfo`,
  `C_Map.GetPlayerMapPosition`, and `C_Container.GetContainerItemQuestInfo` only on clients
  lacking the native global). QA's `/dnetest` can dump this.

## Headline

- **25 distinct modern symbols** are referenced by v1 modules + their Core deps.
- **21 are guaranteed by compat/** (forward/adapter/vendored), of which **~5 are partial/stub**
  (`C_Spell.GetSpellPowerCost`, the three `C_Map.*`, and the no-native fallback branch of
  `C_Container.GetContainerItemQuestInfo`).
- **0 hard dependency on ClassicAPI** — every guaranteed symbol has a vendored fallback; ClassicAPI
  is preferred when loaded purely by the `if not C_X.Fn` guard.
- The remaining grep hits (`C_SpecializationInfo`, `C_EquipmentSet`, `C_NewItems`,
  `C_MerchantFrame`, `C_Reputation`, `C_FriendList`, `C_GuildInfo`, `C_ChatBubbles`,
  `C_ActionBar`, `C_CVar`, `C_AddOns`, `C_UnitAuras`) are **not-needed-v1** — owned by the
  individual panel downports or by ClassicAPI, not by the compat/ shim layer.
